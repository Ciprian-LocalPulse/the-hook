# Contributing

## Contribution Model

THE HOOK is a proprietary defensive security project. This public repository is maintained as a controlled Public Edition for safe product communication, public documentation, and enterprise evaluation.

Public contributions are reviewed selectively and must stay within the Public Edition boundary.

## Acceptable Public Contributions

Acceptable contribution areas include:

- documentation clarity
- proofreading and structure improvements
- public governance file improvements
- public security policy improvements
- accessibility improvements for public materials
- broken-link reports
- safe visual presentation feedback
- public demo polish that does not alter product behavior

## Not Accepted

Do not submit:

- operational deception logic
- telemetry pipelines
- detection rules
- token generation methods
- client environment details
- private playbooks
- prompts or AI orchestration details
- API keys, secrets, credentials, or deployment variables
- customer data or buyer data
- offensive guidance or exploit workflows
- unauthorized targeting guidance
- commercial implementation details
- material copied from a third party without rights

## Pull Request Requirements

Every pull request must:

- describe the public-facing change clearly
- confirm that no secrets or private material were added
- preserve the Public Edition boundary
- preserve proprietary ownership notices
- avoid unsupported security, compliance, or commercial claims
- avoid changing product logic, code behavior, private architecture, or commercial terms

## Public Issues

Use public issues only for:

- public documentation corrections
- broken public links
- public demo defects
- repository hygiene suggestions
- safe product presentation feedback

Do not use public issues to request private source code, restricted implementation details, operational deception methods, telemetry pipelines, detection rules, prompts, customer data, or private enterprise materials.

## Security Reports

Do not report vulnerabilities through public issues or public pull requests.

Use the private reporting process in [SECURITY.md](SECURITY.md).

## Maintainer Discretion

The maintainer may edit, reject, close, or remove submissions that do not align with the defensive purpose, Public Edition boundary, legal posture, procurement readiness, or security expectations of this repository.
