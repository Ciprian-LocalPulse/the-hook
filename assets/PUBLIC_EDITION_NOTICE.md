# Public Edition Notice

## Purpose

This repository is the Public Edition of THE HOOK.

It exists to provide a defensive-security-focused, procurement-safe, public-facing view of the product concept, brand, governance posture, and evaluation surface.

## Public Edition Includes

The Public Edition may include:

- sanitized product overview
- public landing page
- preview assets
- non-sensitive architecture language
- ownership and license notices
- support and security policies
- contribution rules
- public funding configuration

## Public Edition Excludes

The Public Edition does not include:

- operational deception logic
- telemetry pipelines
- detection rules
- client environments
- token generation methods
- private playbooks
- prompts or AI orchestration details
- API keys, secrets, credentials, or deployment variables
- commercial implementation details
- customer data or buyer data
- production deployment instructions
- restricted private enterprise materials

## Private Enterprise Edition

The Private Enterprise Edition is separate from this public repository.

Private package access, private source review, deployment rights, commercial usage rights, implementation support, exclusivity, and support obligations are available only through explicit written agreement.

The public repository should not be treated as the complete product, a complete source package, or a production deployment package.

## Public Discussion Boundary

Public issues and pull requests must remain limited to the public documentation and public preview surface.

Do not request or disclose operational deception logic, telemetry pipelines, detection rules, token generation methods, client environments, private playbooks, prompts, credentials, customer data, internal strategy, or restricted implementation details in public GitHub channels.

## Procurement-Safe Interpretation

This repository may support initial review, but it is not:

- a security audit
- a compliance certification
- a legal opinion
- a production readiness attestation
- a transfer agreement
- an open source release

Formal diligence, security review, legal review, implementation review, or enterprise access review must occur privately under appropriate written terms.
