# Support

This repository is the Public Edition of THE HOOK. Support is limited to public documentation, governance, and safe repository-level questions.

## Public Support

For non-sensitive public questions, you may use GitHub Issues for:

- broken links
- documentation errors
- unclear public positioning
- governance file corrections
- repository metadata or public-readiness feedback

Please do not include security-sensitive information, private implementation details, client information, credentials, screenshots of internal systems, or enterprise deployment assumptions in public issues.

## Commercial Inquiries

For commercial licensing, enterprise evaluation, design-partner discussions, acquisition inquiries, or Private Enterprise Edition access, contact the repository owner through the public GitHub profile or the contact path referenced in the repository README.

Commercial discussions should not be opened as public issues when they involve pricing, private capabilities, implementation details, customer environments, or confidential requirements.

## Security Reporting

Security issues must follow SECURITY.md.

Do not publish vulnerability details, secrets, token examples, private infrastructure references, detection logic, telemetry design, or implementation-sensitive findings in public issues or discussions.

## Public Edition vs. Private Enterprise Edition

Public support covers the Public Edition only. It does not include:

- private deployment support
- client environment review
- telemetry pipeline design
- detection rule development
- deception logic implementation
- private playbooks or prompts
- commercial architecture transfer

Private Enterprise Edition support, if available, requires a separate written agreement.

## Response Expectations

This repository is maintained as a public product and governance surface. Public issue response times are best-effort and depend on severity, relevance, and maintainer availability.

Security reports and commercial inquiries are prioritized through the appropriate private channels.
